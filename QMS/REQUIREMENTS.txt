# Bank Queue Management System - Requirements

## System Requirements

- **Python**: 3.6 or higher
- **Operating System**: Windows, macOS, or Linux
- **RAM**: Minimum 512MB
- **Disk Space**: ~2MB

## Python Dependencies

The Bank QMS uses only Python standard library modules. **No external packages required.**

### Built-in Modules Used:
- `tkinter` - GUI framework (included with Python)
- `collections` - Data structures
- `datetime` - Time handling
- `enum` - Enumerations
- `uuid` - Unique identifiers
- `json` - Data serialization (for exports)

## Installation Steps

### Windows

1. **Download Python** (if not installed)
   - Visit https://www.python.org/downloads/
   - Download Python 3.9 or higher
   - Run the installer
   - ✓ Check "Add Python to PATH"

2. **Verify Installation**
   ```powershell
   python --version
   ```

3. **Run the Application**
   ```powershell
   cd "C:\Users\Diane\OneDrive\Desktop\QMS"
   python main.py
   ```

### macOS

1. **Install Python** (if not installed)
   ```bash
   brew install python3
   ```

2. **Verify Installation**
   ```bash
   python3 --version
   ```

3. **Run the Application**
   ```bash
   cd ~/Desktop/QMS
   python3 main.py
   ```

### Linux

1. **Install Python** (if not installed)
   ```bash
   sudo apt-get install python3 python3-tk
   ```

2. **Verify Installation**
   ```bash
   python3 --version
   ```

3. **Run the Application**
   ```bash
   cd ~/Desktop/QMS
   python3 main.py
   ```

## Troubleshooting Installation

### Issue: "Python not found"
**Solution**: Add Python to PATH and restart terminal

### Issue: "No module named tkinter"
**Solution**: 
- Windows: Reinstall Python and ensure tkinter is selected
- Linux: `sudo apt-get install python3-tk`
- macOS: Should be included by default

### Issue: "Port already in use"
**Solution**: The application doesn't use ports; this shouldn't occur

### Issue: GUI doesn't display
**Solution**: Ensure you're running in a desktop environment (not SSH)

## Performance Notes

- **Small System**: 3-5 counters, 20-50 customers
- **Medium System**: 5-10 counters, 50-200 customers
- **Large System**: 10+ counters, 200-500 customers

Beyond 500 customers, consider:
- Archiving completed customer records
- Running multiple independent systems
- Optimizing the queue algorithm

## Support & Troubleshooting

### Application Won't Start
1. Verify Python installation: `python --version`
2. Check file permissions
3. Try running test script first: `python test.py`
4. Check console for error messages

### GUI Issues
- Restart the application
- Check for conflicting display settings
- Try different zoom levels (if on high DPI display)

### Performance Issues
- Reduce number of display items
- Close other applications
- Check available RAM
